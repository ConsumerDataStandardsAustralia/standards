V1.33.0 Release Notes

Release notes for version 1.33.0 of the CDR Standards.

Changes Made

Change Requests

This release addresses the following minor defects raised on Standards Staging:

  • Standards Staging #429 - Refer to components in Banking API spec
  • Standards Staging #431 - Fix spelling, grammar and punctuation
  • Standards Staging #435 - Maintenance Iteration 21 - typos and formatting
  • Standards Staging #442 - Apply consistent styling to the Common Field Types table
  • Standards Staging #443 - Apply consistent styling in the Transaction Security section
  • Standards Staging #463 - Remove redundant generated 'additional' file
  • Standards Staging #468 - Update DSB Logo and links

This release addresses the following change requests raised on Standards Maintenance:

  • Standards Maintenance #473 - Add RFC8174 to list of normative references and update the use of Requirements Levels
  • Standards Maintenance #654 - Clarify Transaction Security requirements
  • Standards Maintenance #655 - Get Metrics V5 error metrics documentation
  • Standards Maintenance #657 - Addition of LVR in the enumerated values list for constraintType
  • Standards Maintenance #661 - Obligation milestones for 2026 and 2027
  • Standards Maintenance #663 - Maintenance Iteration 21 Holistic Feedback
  • Standards Maintenance #664 - New Enums for Voluntary disclosure of additional service overlays
  • Standards Maintenance #667 - Clean up of Refresh Token requirements
  • Standards Maintenance #675 - PAR/RFC9126 in Normative references appears twice
  • Standards Maintenance #666 - Retirement of OIDC Hybrid Flow

Decision Proposals

This release addresses the following Decision Proposals published on Standards:

  • Decision Proposal #350 - August 2024 Rules - Standards Impacts
  • Decision Proposal #356 - Maintenance Iteration 21
