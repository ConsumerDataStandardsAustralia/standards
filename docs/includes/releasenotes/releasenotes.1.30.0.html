V1.30.0 Release Notes

Release notes for version v1.30.0 of the CDR Standards.

Changes Made

Change Requests

This release addresses the following minor defects raised on Standards Staging:

  • Standards Staging #310 - Review date format conventions
  • Standards Staging #361 - Typo in sector_identifier_uri description
  • Standards Staging #362 - Update PPID link
  • Standards Staging #370 - Add 'Consumer Experience Principles' to the Introduction text in the Principles section of the CDS and add CX acronym
  • Standards Staging #371 - Avoid scrollbars for long lines
  • Standards Staging #376 - Custom field types for Response Headers are not displayed
  • Standards Staging #388 - Fix typo in Amending Authorisation Standards: Changing Attributes

This release addresses the following change requests raised on Standards Maintenance:

  • Standards Maintenance #543 - refresh_token_expires_at and sharing_expires_at claims listed as MUST be supported
  • Standards Maintenance #624 - Improved structure for Solar Time Varying Tariffs
  • Standards Maintenance #625 - Additional field to support Step Tariff calculations
  • Standards Maintenance #627 - EnergyPlanTariffPeriod - Change to daily supply charge
  • Standards Maintenance #629 - Maintenance Iteration 18 Holistic Feedback
  • Standards Maintenance #631 - Updates to 'Revoking consent' Standards
  • Standards Maintenance #632 - Concurrent consent support and cdr_arrangement_id

Decision Proposals

This release addresses the following Decision Proposals published on Standards:

  • Decision Proposal #340 - Maintenance Iteration 18

API Endpoints

Change Description Link
Banded Daily Supply Charges Standards Maintenance #627: Made following changes to EnergyPlanTariffPeriod to allow sharing of banded daily supply charges:
  • Added dailySupplyChargeType - new ENUM field to note if the daily supply charge is single or banded
  • Added bandedDailySupplyCharges- A new object allowing representation of banded daily supply charges
  • Changed dailySupplyCharges to dailySupplyCharge
 Energy APIs
Support Step Tariff calculations Standards Maintenance #625: Added new optional period field to the following to help support stepped tariff calculations:
  • EnergyPlanControlledLoad.singleRate
  • EnergyPlanSolarFeedInTariffV2.singleRate
  • EnergyPlanControlledLoad.timeOfUseRates
  • EnergyPlanSolarFeedInTariffV2.timeVaryingTariffs
  • EnergyPlanTariffPeriod.timeOfUseRates
 Energy APIs
Improved structure for Solar Time Varying Tariffs Standards Maintenance #624:
  • Converted solarFeedInTariff.timeVaringTariffs into an array to allow representation of multiple feed in tariffs
  • Added new mandatory displayName field to solarFeedInTariff.timeVaryingTariffs
  • Added new ENUM values CURRENT and VARIABLE to solarFeedInTariff.scheme
 Energy APIs
