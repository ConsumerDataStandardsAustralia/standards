API Endpoints

Change Description Link
Updates to Banking Products and Accounts Decision Proposal #338: Incorporated changes from Decision 338:
  • Incremented endpoint versions:
    • Get Products v4
    • Get Product Detail v6
    • Get Account Detail v4
  • Changes incorporated into the updated versions include:
    • Added cardScheme and cardType fields in the existing cardArt object
    • Added BALANCE_TRANSFER lendingRateType option
    • Modified the feature isActivated field behaviour
    • Added FUNDS_AVAILABLE_AFTER featureType option
    • Added OTHER constraintType option
    • Clarified the descriptions of the constraints and features fields
    • Updated and aligned the PENSION_RECIPIENT and STUDENT eligibility and discount eligibility 'Use of additionalValue Field' details
    • Added OTHER and UNCONSTRAINED repaymentType options and specified the field as mandatory
    • Added OTHER and UNCONSTRAINED loanPurpose options and specified the field as mandatory
    • Modified the existing applicabilityConditions structure in rate tier objects and added the same structure at the rate level
    • Added rate applicationType field in rate objects and modified the corresponding applicationFrequency field requirement
    • Changed the field type for the minimumValue and maximumValue fields in the rate tier structure from 'Number' to 'string'
    • Removed the VARIABLE feeType value, added eight new feeType values, and refined the fee and fee discount structures.
  • Minor documentation updates for consistency.
 Banking APIs
LCCD changes Decision Proposal #361:
  • Incremented versions of Get Service Points, Get Service Point Detail, Get Service Points (SR) and Get Service Point Detail (SR) APIs to v2
  • Added new field EnergyServicePointV2.lastConsumerChangeDate.
